Hermes Agent 搭建教程：从零到微信 AI 助手

手把手教你用一台服务器/电脑，部署属于你自己的 AI 助手，并接入微信随时聊天。

写在前面

Hermes Agent 是一个开源的个人 AI 助手框架，相比直接用 ChatGPT/Claude 网页版，它的优势在于：

  • 🤖 可接入微信 — 用微信就能跟 AI 对话，不用来回切网页
  • 🧠 自带知识库 — 能记住你的偏好，跨会话持久记忆
  • 🔧 可定制 — 工具链、技能、角色人设都能自己改
  • 🔒 数据在自己手里 — 部署在你自己的服务器上，隐私性更好

如果你跟我一样，想要一个能随时在微信上聊、帮你查资料、记事情、分析问题的 AI 助手，这篇教程就是为你准备的。

环境要求

项目 要求
操作系统 Linux / macOS / Windows (WSL2)
网络 能访问 GitHub/国内镜像
空间 至少 500MB 可用
推荐 一台长期在线的云服务器（京东云/阿里云/腾讯云轻量应用服务器即可）

💡 如果只是体验测试，Windows WSL2 或 Mac 本地跑也可以，但关掉终端服务就停了。

第一步：创建独立用户（推荐）

Hermes 会将技能文档、记忆数据存在本地。如果多个用户共用一个账号跑多个 Gateway，会导致数据互相可见。

⚠️ 建议为每个实例创建一个独立的系统用户，避免数据混淆。

创建完成后，切换到新用户继续操作：

常见问题

Q：安装时提示 curl 命令找不到？

Q：微信扫码后提示链接已过期？
重新运行 hermes gateway setup 生成新的二维码链接即可。

Q：想换一个 AI 模型？

会重新进入配置向导，可以修改 API Provider 和模型。

Q：如何让 Hermes 记住我的信息？
直接跟它说 “记住……”，它会自动存入记忆，跨会话持久保存。

Q：能接多个微信账号吗？
可以。创建不同的系统用户，每个用户跑一个独立的 Hermes 实例和 Gateway，互不干扰。

进阶玩法（可以自己去探索）

  • 📚 Skill — 给 Hermes 写自定义技能，比如定时推送新闻、自动查天气
  • 🧠 Memory — 让它长期记住你的偏好、项目状态
  • 🔧 Tools — 扩展工具链，让它能操作你的服务器、查数据库
  • 🎤 语音 — 配置 TTS，Hermes 可以用语音回复

写在最后

部署完 Hermes 之后，它就变成一个随时在线的 AI 助手。从查资料、分析问题、写代码到日常闲聊，都可以在微信上完成。

这个项目完全开源，社区也在持续更新。如果遇到问题，可以去 Hermes Agent 中文社区 看看。

教程里的截图我会后续补上，如果你跟着搭遇到了坑，欢迎留言交流 🙌
